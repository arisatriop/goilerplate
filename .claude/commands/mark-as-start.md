# Mark as Start — Post Jira Ticket Comment

Post a "starting work" comment to a Jira ticket when you begin working on it.

Usage:
- `/mark-as-start TICKET_ID` — post directly to that ticket
- `/mark-as-start` — fetch tickets assigned to you, then pick one

1. If no TICKET_ID was given:
   - Use the `mcp__jira__search_jira_issues` tool with JQL:
     ```
     assignee=currentUser() ORDER BY updated DESC
     ```
     Request fields: `summary,status,priority`, maxResults: 20.
   - Display the results as a numbered list in this format:
     ```
     #  TICKET-ID  [Status] Priority — Summary
     1. PROJ-42    [To Do]  Medium   — Add user profile endpoint
     2. PROJ-38    [In Progress] High — Fix auth token expiry
     ...
     ```
   - Ask the user: "Which ticket are you starting? (enter number or ticket ID)". Wait for their answer, then set TICKET_ID accordingly.

2. Get the current timestamp:
   ```bash
   date "+%Y-%m-%d %H:%M:%S"
   ```

3. Compose the comment text:
   ```
   🚀 Starting work — [timestamp]

   ---
   Generated by Claude (https://claude.ai/code)
   ```
   Use the actual timestamp from step 2.

4. Post the comment using the `mcp__jira__add_jira_comment` tool with:
   - `issue_key`: TICKET_ID
   - `comment`: the composed text from step 3

5. Confirm to the user: "Comment posted to $TICKET_ID."
