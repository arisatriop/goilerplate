# Mark as Start — Post Jira Ticket Comment

Post a "starting work" comment to a Jira ticket when you begin working on it.

Usage:
- `/mark-as-start TICKET_ID` — post directly to that ticket
- `/mark-as-start` — list tickets assigned to you, then pick one

Jira credentials (`JIRA_URL`, `JIRA_EMAIL`, `JIRA_API_TOKEN`) are read from
`config/.env` by the helper scripts in `.claude/scripts/`.

## 1. Resolve the ticket

If a TICKET_ID was given, use it. Otherwise list assigned tickets and ask which one:
```bash
.claude/scripts/jira-tickets.sh
```
Ask: "Which ticket are you starting? (enter number or ticket ID)". Map the chosen
number to the ticket ID from the table.

## 2. Compose and post the comment

Get the current timestamp:
```bash
date "+%Y-%m-%d %H:%M:%S"
```

Build the ADF document and pipe it into `jira-comment.sh` (substitute the real
timestamp and ticket ID):
```bash
echo '{
  "version": 1,
  "type": "doc",
  "content": [
    {"type": "paragraph", "content": [
      {"type": "text", "text": "🚀 Starting work — <TIMESTAMP>"}
    ]},
    {"type": "rule"},
    {"type": "paragraph", "content": [
      {"type": "text", "text": "Generated by "},
      {"type": "text", "text": "Claude", "marks": [{"type": "link", "attrs": {"href": "https://claude.ai/code"}}]}
    ]}
  ]
}' | .claude/scripts/jira-comment.sh <TICKET_ID>
```

The script prints `Comment posted to <TICKET_ID>.` on success or a clear error
otherwise — relay its output. Do not ask for confirmation before posting.
